建立联系

1、 Git下载地址

（小乌龟可以不安装，没什么用，使用git bash 就行）
3.1添加ssh-key
在gitlab完成注册后，首先访问这里，向你的帐户添加ssh keys，ssh keys的生成可以参考如下的步骤，打开2.2安装的Git Bash，输入如下命令(自行替换其中的用户名和邮箱信息)

git config –global user.name “(你的名字）”
git config –global user.email “(你的邮箱)”
ssh-keygen -t rsa -C (你的邮箱)
提示输入密码可以不用写，直接回车
最终生成的文件一般位于C:\Users\UserName.ssh文件夹下，打开id_rsa.pub（可以使用万能的vs code），获取通信
复制ssh，添加到gitlab中User Settings>SSH Keys

与远程仓库建立连接：git remote add origin XXXXX.git

获取项目：

使用git branch 查看本地是否具有dev分支(开发中不会直接提交到master分支，公司远端有dev和master分支，代码都提交到dev分支)
如果没有 git fetch origin dev
git checkout -b dev origin/dev在本地创建分支dev并切换到该分支
git pull origin dev就可以把gitLab上dev分支上的内容都拉取到本地了
再次查询本地是否存在dev分支，还需存在master分支，git branch
如果不存在master分支，执行命令 git checkout -b master
切换到dev分支，git checkout dev
此时最新的代码就已经拉取到本地，并且本地使用的分支为dev分支

常用操作

git bash 常用指令(注意空格)：

分支操作：

git branch 查看分支
git branch （分支名字）创建分支
git branch -d (分支名字) 删除分支
git checkout （分支名字）切换其他分支
git checkout -b（分支名字）创建并切换其他分支
git merge (分支名字) 与（分支名字）合并
git

上传：

git add . 添加全部文件
git add (文件) （文件）添加文件
git add (文件夹) 添加文件夹
git commit -m “(描述)” 为缓存内的文件进行描述
git push 上传到远程分支（当前分支）
git push origin (分支名字) 上传到(分支名字)分支

下拉：

Git clone (项目地址)
git pull 拉取当前的远程分支
git pull origin （分支名字）拉取分支并
git pull origin （远程分支）:（本地分支）拉取远程分支与本地分支合并

回退：

git add 后撤销：

git reset HEAD . 撤销所有add文件
git reset HEAD -文件名字e撤销单个add文件

git commit 后撤销：

git reset –soft head 只回退commit的信息，保留修改代码：
git reset –hard head^ 彻底回退到上次commit版本，不保留修改代码：
说明：
HEAD ：当前版本
HEAD^ ：上一个版本
–hard 参数会抛弃当前工作区的修改
–soft 参数的话会回退到之前的版本，但是保留当前工作区的修改，可以重新提交
git checkout . 撤销所有本地改动代码：
git reset –hard （远程分支名）本地代码回退到与git远程仓库保持一致

git push撤销：

回滚此次push到服务器的代码：
git log查看commit的信息
git revert 以前commit的id
git push 此时本地回滚的代码到服务器就可以了

git merge 撤销：

$ git checkout 【行merge操作时所在的分支】
$ git reset –hard 【merge前的版本号】
